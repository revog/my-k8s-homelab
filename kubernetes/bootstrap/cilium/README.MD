# Cilium Bootstrap (Day-0)
This directory contains a **pre-rendered Cilium manifest** used for initial cluster bootstrapping via Talos configuration (`cluster.network.cni.urls`).
Cilium is installed at a very early state (before Flux exists), therefore Helm/Kustomize controllers cannot be used at this stage.

## Create pre-templated Cilium manifests for Cluster bootstrapping
The bootstrap manifest must be generated using Helm, ensuring: 
- CRDs are included
- Helm rendering is accurate
- No dependency on runtime controllers

```bash
# Overwrite file with a clear header
echo "# This manifest was generated by automation. DO NOT EDIT." > kubernetes/bootstrap/cilium/cilium.yaml

# Render Cilium deployment with Helm
helm template cilium oci://quay.io/cilium/charts/cilium \
  --version 1.19.3 \
  --namespace kube-system \
  --include-crds \
  -f kubernetes/applications/infra/cilium/app/values.yaml \
  > kubernetes/bootstrap/cilium/cilium.yaml
```

⚠️ This pre-rendered file is only used during cluster bootstrapping process (day-0). After successful deployment of Flux, application will be maintained through Flux mechanisms.
